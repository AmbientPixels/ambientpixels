CI/CD Pipeline

Local dev in C:\_projects\AmbientPixels.ai\—Dreamweaver or IDE—flows to GitHub’s main branch with standardized commits (git commit -m "Feature - vX.X.X"—low-risk).

  • GitHub Actions: static.yml deploys static content—api.yml for future APIs
  • Triggers: Push to main or PR merge—Parallel workflows
  • Secrets: AZURE_CREDENTIALS via GitHub Secrets—Secure

Azure Services

Delivery Process

Staged in batches—incremental and stable:

  1. Batch 1: test.html—Core structure—PixelPulse confirms
  2. Batch 2: CSS files (base.css, layout.css, etc.)—Styles—PixelPulse confirms
  3. Batch 3: JS files (theme.js, main.js, etc.)—Functionality—PixelPulse confirms
  4. Batch 4: Incremental features (banners, modals)—PixelPulse confirms
  5. Final: Polish—Ready for Grok Deeper—Celebrate!

Batch delivery—low-risk, iterative—keeps the grid humming.

Security Grid

API security—token-based (JWT), CORS locked to ambientpixels.ai—future-proof, low-risk. Workflows split—static.yml and api.yml—isolated and secure. Credentials in GitHub Secrets—locked tight.

Tech Stack

Frontend: HTML5, CSS3, Vanilla JS—Font Awesome 6.5.1 icons. Hosted on Azure Static Web App, with future Node.js serverless via Function App. GitHub VCS—main branch, .github/workflows/. Tools: Dreamweaver, VS Code compatible—lean and mean.

Lessons Learned

“Glitches forge strength—code evolves.”
  • Mobile Menu: Toggle failed—Fixed with nav.js—Parent .nav > child .nav-links
  • Toggles: Grid broke—Resolved with data-toggle—Parent
    > child .content
  • Hero Images: theme.css override (.transition-style { opacity: 0; })—Fixed with .hero-slide.active { opacity: 1 !important; }
  • Loading: Icon/text clash—Z-index (loading: 4, text: 3, slides: 1-2)—2s timing

Design Considerations

Light, modular code—split CSS/JS, minimal dependencies—extends via components.css. Clean hierarchy—.hero > .hero-slides > .hero-slide, .nav > .nav-links. Modern design—cyberpunk glow, light/dark themes—flexible template with icon-only nav, toggle-ready grid, static hero headlines—future flair (e.g., bounce) ready—low-risk.
