Pipeline

CI/CD Workflow

Our process is an automated CI/CD pipeline:

  • Dreamweaver (DW) Edit: Local edits to files (e.g., index.html, _images/styles.css) in Adobe Dreamweaver.
  • Git Push: Commit changes to local repo (C:\_projects\AmbientPixels.ai) and push to origin/master on GitHub (github.com/AmbientPixels/ambientpixels).
  • GitHub Workflows: Two workflows trigger:
    • azure-static-web-apps-calm-sky-05cc8e110.yml: Deploys static content to Azure Static Web App.
    • master_ambientpixels-meme-api-fn.yml: Deploys API updates to Azure Function App.
  • Azure Deployment: Updates go live:
    • Static Web App (ambientpixelsV2)—https://calm-sky-05cc8e110.6.azurestaticapps.net/ and ambientpixels.ai.
    • Function App (ambientpixels-meme-api-fn)—API endpoints (e.g., https://ambientpixels-meme-api-fn.azurewebsites.net/api/getToken).

Why Fast? Pre-configured GitHub Actions with API tokens—Push triggers instant deployment without manual steps.

Azure Services

We run two Azure services side-by-side:

  • Static Web App (ambientpixelsV2):

    Purpose: Hosts static content—HTML, CSS, JS (e.g., index.html, _images/).

    URL: https://calm-sky-05cc8e110.6.azurestaticapps.net/—Custom domain ambientpixels.ai.

    Deployment: GitHub workflow azure-static-web-apps-calm-sky-05cc8e110.yml uploads files directly—no build step (skip_app_build: "true").

    SSL: Azure-managed HTTPS—Validated with TXT record.

  • Function App (ambientpixels-meme-api-fn):

    Purpose: Runs serverless APIs—getToken (UUIDs), memeGenerator (memes).

    URL: https://ambientpixels-meme-api-fn.azurewebsites.net/api/[endpoint].

    Deployment: GitHub workflow master_ambientpixels-meme-api-fn.yml builds Node.js functions (e.g., getToken/index.js).

    CORS: Secured with Access-Control-Allow-Origin—Allows calm-sky… and ambientpixels.ai.

Integration: Static Web App serves frontend, fetching from Function App APIs via scripts.js. Separation ensures lightweight static content and scalable APIs.

Dual GitHub Workflows

Why Two Workflows?

  • Static Content: azure-static-web-apps-calm-sky-05cc8e110.yml deploys HTML/CSS/JS—Triggers on all master pushes.
  • API Functions: master_ambientpixels-meme-api-fn.yml deploys Node.js functions—Triggers on API file changes (e.g., getToken/, meme-api/, package.json).
  • Separation: Prevents unnecessary API deploys—Optimizes speed.

Static Workflow:

on:
  push:
    branches: [ "master" ]
jobs:
  build_and_deploy_job:
    steps:
      - uses: Azure/static-web-apps-deploy@v1
        with:
          azure_static_web_apps_api_token: ${{ secrets.AZURE_STATIC_WEB_APPS_API_TOKEN_CALM_SKY_05CC8E110 }}
          app_location: "/"
          output_location: ""
          skip_app_build: "true"

Function Workflow:

on:
  push:
    branches: [ "master" ]
    paths:
      - 'getToken/**'
      - 'meme-api/**'
      - 'package.json'
jobs:
  build-and-deploy:
    steps:
      - uses: Azure/functions-action@v1
        with:
          app-name: 'ambientpixels-meme-api-fn'
          package: '.'
          publish-profile: ${{ secrets.AZUREAPPSERVICE_PUBLISHPROFILE_E5BE881F58F440D69430618422EEB336 }}

Process: Single git push—GitHub splits work—Static deploys always—Function deploys conditionally.

Scripts

Core Scripts

Purpose: _images/scripts.js powers shared functionality across pages—UI toggles, version display, meme generation.

Key Functions:

  • Version Number: Sets dynamic version—const VERSION_NUMBER = "v0.17.1-20250328";—Updates #version-number.
  • UI Toggles: toggleTheme, toggleMenu, toggleSection, toggleGroup, toggleAllSections—Handle light/dark, mobile nav, and section collapsing.
  • Animations: animateSections—Fades in sections on load.
  • Meme Generator: getAccessToken, generateMeme—Fetches API token, generates memes with prompt.
  • Theme Switcher: setTheme—Sets light/dark mode explicitly (v0.17.1-20250328).
  • Chaos Mode: toggleChaosMode—Toggles chaotic effects (v0.17.1-20250328).

Code Example:

async function generateMeme() {
    const promptInput = document.getElementById('memePrompt');
    const defaultPrompt = 'A confused robot in a disco';
    const prompt = promptInput?.value.trim() || defaultPrompt;
    const generateBtn = document.getElementById('memeGenerateBtn');
    const loadingDiv = document.getElementById('memeLoading');
    const memeImage = document.getElementById('memeImage');
    const topText = document.getElementById('memeTopText');
    const bottomText = document.getElementById('memeBottomText');

    if (!generateBtn || !loadingDiv || !memeImage || !topText || !bottomText) return;

    generateBtn.disabled = true;
    loadingDiv.classList.add('visible');
    memeImage.classList.remove('visible');
    topText.innerText = '';
    bottomText.innerText = '';

    try {
        const token = await getAccessToken();
        if (!token) throw new Error('No access token available');
        const response = await fetch(API_ENDPOINT, {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json',
                'Authorization': `Bearer ${token}`
            },
            body: JSON.stringify({ endpoint: "image", prompt: prompt })
        });
        if (!response.ok) throw new Error(`API error: ${response.status}`);
        const blob = await response.blob();
        memeImage.src = URL.createObjectURL(blob);
        memeImage.classList.add('visible');
        topText.innerText = prompt.slice(0, 20);
        bottomText.innerText = prompt.slice(-20);
    } catch (error) {
        console.error('Meme generation error:', error);
        memeImage.src = 'https://placehold.co/300x300?text=API+Error';
        topText.innerText = 'OOPS';
        bottomText.innerText = 'API BROKE';
        memeImage.classList.add('visible');
    } finally {
        loadingDiv.classList.remove('visible');
        generateBtn.disabled = false;
    }
}

Hero Randomizer

Purpose: Adds chaotic flair to index.html—Randomizes hero banner images and taglines on load.

Location: _images/hero-randomizer.js—Split from scripts.js in v0.16.8-20250328 for homepage-specific use.

Functionality:

  • Taglines: Picks from 7 chaotic phrases (e.g., “Pixel storm incoming—duck!”)—Displays in #randomTagline with letter-by-letter animation.
  • Images: Randomizes 11 banners (banner.jpg, banner1.jpg to banner10.jpg)—Sets as .hero background.
  • Loading: Shows #loadingScreen for 1.5s—Hides, reveals tagline.
  • Execution: Wrapped in DOMContentLoaded—Ensures DOM readiness.

Code:

document.addEventListener('DOMContentLoaded', () => {
    const taglines = [
        "Oops, the pixels got drunk again!",
        "Chaos.exe has entered the chat.",
        // ... 5 more ...
    ];
    const taglineElement = document.getElementById('randomTagline');
    const loadingScreen = document.getElementById('loadingScreen');
    if (taglineElement && loadingScreen) {
        const randomTagline = taglines[Math.floor(Math.random() * taglines.length)];
        taglineElement.innerHTML = randomTagline.split('').map(char =>
            char === ' ' ? `[span] [/span]` : `[span]${char}[/span]`
        ).join('');
        setTimeout(() => {
            loadingScreen.style.display = 'none';
            taglineElement.style.display = 'block';
        }, 1500);
    }

    const heroImages = [
        '_images/banner.jpg',
        '_images/banner1.jpg',
        '_images/banner2.jpg', // Replaced v0.16.7
        '_images/banner3.jpg',
        '_images/banner4.jpg',
        '_images/banner5.jpg',
        '_images/banner6.jpg', // Added v0.16.7
        '_images/banner7.jpg',
        '_images/banner8.jpg',
        '_images/banner9.jpg',
        '_images/banner10.jpg'
    ];
    const heroElement = document.querySelector('.hero');
    if (heroElement) {
        const randomHeroImage = heroImages[Math.floor(Math.random() * heroImages.length)];
        heroElement.style.backgroundImage = `url('${randomHeroImage}')`;
    }
});

UI Functions

toggleSection: Collapses/expands sections—Click headers.

function toggleSection(header) {
    const content = header.nextElementSibling;
    content.classList.toggle('active');
}

toggleGroup: Collapses/expands section groups—Swaps chevron icons.

function toggleGroup(header) {
    const content = header.nextElementSibling;
    content.classList.toggle('active');
    const icon = header.querySelector('.toggle-icon');
    icon.classList.toggle('fa-chevron-down');
    icon.classList.toggle('fa-chevron-up');
}

toggleAllSections: Toggles all groups and sections—Swaps expand/compress icons.

toggleMenu: Mobile navbar toggle—Burger icon.

Troubleshooting

Purpose: Quick fixes for common issues—Educates on debugging.

  • Images Not Loading: Check paths (e.g., _images/bannerX.jpg), case sensitivity, GitHub repo sync. Fix: Re-upload, verify hash.
  • Scripts Fail: Ensure defer on [script] tags, use DOMContentLoaded. Log errors with console.log.
  • Styles Misaligned: Verify max-width, margin consistency—e.g., .group-header vs .section-container.
  • API Errors: Check CORS, token fetch—Test endpoints in browser (https://ambientpixels-meme-api-fn.azurewebsites.net/api/getToken).
  • Footer Missing: Check for unescaped < or > in code blocks—Replace with [ and ] (fixed in v0.17.1-20250328).
  • Chaos Mode Stalls: Ensure toggleChaosMode toggles .chaos-active—Check browser console for JS errors.

Tip: Open browser console (F12)—Look for 404s, JS errors, or CSS warnings.
