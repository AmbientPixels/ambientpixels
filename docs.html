Overview

Your Technical Blueprint for Ambient Pixels

This document is the definitive guide for ambientpixels.ai—covering our CI/CD pipeline, security protocols, technical stack, scripts, and style system as of v0.17.4-20250329 (March 29, 2025, 12:00 PM PDT). It’s designed to onboard and train contributors on deployment workflows, code structure, and styling conventions.

Pipeline

CI/CD Workflow

Our CI/CD pipeline automates deployment:

  • Local Development: Edit in Adobe Dreamweaver—HTML, CSS, JS files in C:\_projects\AmbientPixels.ai\.
  • Version Control: Commit changes to GitHub repo—branch main.
  • GitHub Actions:
    • static.yml: Builds and deploys static content to Azure Static Web App.
    • api.yml: Deploys API functions to Azure Function App.
  • Triggers: Push to main or PR merge—workflows run in parallel.

Example commit: git commit -m "Update grid layout - v0.17.4"

Azure Services

We leverage Azure for hosting and serverless functionality:

  • Static Web App: Hosts index.html, docs.html, CSS/JS in _images/.
  • Function App: Runs serverless APIs—e.g., memeGenerator endpoint at https://ambientpixels-meme-api-fn.azurewebsites.net/api/.
  • Configuration: Managed via Azure Portal—SWA for static, Function App for APIs with token auth.

Security

Security is baked into our pipeline and deployment:

  • API Security:
    • Token-based auth—JWTs issued via getToken endpoint.
    • CORS configured—allowed origins: ambientpixels.ai.
  • Dual Workflows:
    • Static: .github/workflows/static.yml—isolates HTML/CSS/JS.
    • API: .github/workflows/api.yml—secures function deployment.
  • Secrets: Stored in GitHub Secrets—e.g., AZURE_CREDENTIALS.

Tech Stack

Our technical foundation:

  • Frontend: HTML5, CSS3, vanilla JS—Font Awesome 6.5.1 for icons.
  • Hosting: Azure Static Web App—ambientpixels.ai.
  • Serverless: Azure Function App—Node.js runtime for APIs.
  • VCS: GitHub—main branch, .github/workflows/.
  • Tools: Adobe Dreamweaver—local dev environment.

Troubleshooting

Common issues and fixes:

  • API Errors: Verify token in Function App logs—GET /api/getToken.
  • Style Not Applied: Clear browser cache (Ctrl+F5), check _images/ links.
  • Workflow Fails: Inspect GitHub Actions logs—ensure secrets are set.
