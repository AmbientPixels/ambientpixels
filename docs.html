CI/CD Pipeline

Publishing Workflow

Chaos Meets Automation

Our CI/CD pipeline automates deployment:

  • Dreamweaver (DW) Edit: Edit files locally (e.g., projects.html, _images/styles.css) in Adobe Dreamweaver.
  • Git Push: Commit to C:\_projects\AmbientPixels.ai, push to origin/master on GitHub (repo).
  • GitHub Workflows: Two workflows trigger:
    • azure-static-web-apps-calm-sky-05cc8e110.yml: Deploys static content to Azure Static Web App.
    • master_ambientpixels-meme-api-fn.yml: Deploys API updates to Azure Function App.
  • Azure Deployment: Updates go live:
    • Static Web App (ambientpixelsV2)—https://calm-sky-05cc8e110.6.azurestaticapps.net/ and ambientpixels.ai.
    • Function App (ambientpixels-meme-api-fn)—API endpoints (e.g., https://ambientpixels-meme-api-fn.azurewebsites.net/api/getToken).

Why Fast? Pre-configured GitHub Actions with API tokens—Push triggers instant deployment.

Azure Services Setup

Two Azure services power the site:

  • Static Web App (ambientpixelsV2):

    Purpose: Hosts static content—HTML, CSS, JS (e.g., index.html, projects.html, _images/).

    URL: https://calm-sky-05cc8e110.6.azurestaticapps.net/—Custom domain ambientpixels.ai.

    Deployment: Workflow azure-static-web-apps-calm-sky-05cc8e110.yml—No build (skip_app_build: "true").

    SSL: Azure-managed HTTPS—Validated with TXT record.

  • Function App (ambientpixels-meme-api-fn):

    Purpose: Runs serverless APIs—getToken (UUIDs), memeGenerator (memes).

    URL: https://ambientpixels-meme-api-fn.azurewebsites.net/api/[endpoint].

    Deployment: Workflow master_ambientpixels-meme-api-fn.yml—Builds Node.js functions.

    CORS: Allows calm-sky… and ambientpixels.ai.

Integration: Static frontend calls Function App APIs via fetch—keeps content lightweight, APIs scalable.

API Security

Token-Based: getToken generates UUIDs—Future memeGenerator validation planned.

CORS: Configured in getToken/index.js:

headers: {
    "Access-Control-Allow-Origin": "https://calm-sky-05cc8e110.6.azurestaticapps.net",
    "Access-Control-Allow-Methods": "GET, POST, OPTIONS",
    "Access-Control-Allow-Headers": "Content-Type, Authorization"
}

Also set in Azure Portal > Function App > CORS for ambientpixels.ai and www.ambientpixels.ai.

Status: Secured—Scalable for future domains.

GitHub Workflows

Dual Workflows:

  • Static: azure-static-web-apps-calm-sky-05cc8e110.yml—All master pushes.
  • API: master_ambientpixels-meme-api-fn.yml—API file changes only (e.g., getToken/).
  • Separation optimizes speed—Static deploys instant, API builds conditional.

Static Workflow

on:
  push:
    branches: [ "master" ]
jobs:
  build_and_deploy_job:
    steps:
      - uses: Azure/static-web-apps-deploy@v1
        with:
          azure_static_web_apps_api_token: ${{ secrets.AZURE_STATIC_WEB_APPS_API_TOKEN_CALM_SKY_05CC8E110 }}
          app_location: "/"
          output_location: ""
          skip_app_build: "true"

Function Workflow

on:
  push:
    branches: [ "master" ]
    paths:
      - 'getToken/**'
      - 'meme-api/**'
      - 'package.json'
jobs:
  build-and-deploy:
    steps:
      - uses: Azure/functions-action@v1
        with:
          app-name: 'ambientpixels-meme-api-fn'
          package: '.'
          publish-profile: ${{ secrets.AZUREAPPSERVICE_PUBLISHPROFILE_E5BE881F58F440D69430618422EEB336 }}

Push: Single git push splits work—Static always, Function conditional.

Style Guide

Styles

Chaos Unleashed in Style

Our design system (V16.4.3) powers ambientpixels.ai. Here’s the suite:

Structure

  • Standalone Sections: Like #intro—Static, always visible—<section class="section-container">.
  • Section Groups: Collapsible groups (e.g., “CI/CD Pipeline”)—<div class="section-group"> with nested .section-container.
  • Toggle: Click .group-header or h2—.active class shows content.

Text Styles

Strong: Bold—<strong>.

Highlight: Color pop—<span class="highlight">.

Paragraphs flow naturally.

  • Bullet—<ul><li>—Indented 25px.
  • Bullet with Strong: Emphasis.
  • Bullet with Highlight: Flair.

Callouts

Warning: Critical—.callout.callout-warning.

Info: Details—.callout.callout-info.

Highlight Box: Key points—.highlight-box.

Headings

H3: Bold—<h3>

H4: Smaller—<h4>

H5: Tiny—<h5>

Code

const chaos = "Ambient Pixels";
console.log(chaos + " rocks!");

Inline: let x = 42;—<code>.

Links & Buttons

Standard—.link-standard.

Hover—.link-hover.

—.btn.btn-primary, .btn.btn-secondary.

Special

“Chaos is creativity.”—.blockquote

Small text—.small-text.

Drop cap—.drop-cap.

Functions

  • toggleSection: Toggles .section-content—onclick="toggleSection(this)".
  • toggleGroup: Toggles .group-content—onclick="toggleGroup(this)", swaps chevron.
  • toggleAllSections: Toggles all sections—#toggleAllSections, swaps expand/compress icon.
  • toggleMenu: Mobile menu—.navbar-toggle.
  • toggleTheme: Light/dark—.theme-toggle.
  • backToTop: Scrolls top—.back-to-top, appears on scroll.

Load State: Add .active to .group-content or .section-content for uncollapsed on load—JS can set dynamically (e.g., first section).

Features

  • Responsive—Mobile adjusts margins (15px).
  • Theme switch—body.light toggles colors.
  • Future-ready—.loading shimmer for dynamic content.
