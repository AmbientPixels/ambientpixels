Pipeline

CI/CD Workflow

Our process is an automated CI/CD pipeline:

  • Dreamweaver (DW) Edit: Local edits to files (e.g., template.html, _images/styles.css) in Adobe Dreamweaver.
  • Git Push: Commit changes to local repo (C:\_projects\AmbientPixels.ai) and push to origin/master on GitHub (github.com/AmbientPixels/ambientpixels).
  • GitHub Workflows: Two workflows trigger:
    • azure-static-web-apps-calm-sky-05cc8e110.yml: Deploys static content to Azure Static Web App.
    • master_ambientpixels-meme-api-fn.yml: Deploys API updates to Azure Function App.
  • Azure Deployment: Updates go live:
    • Static Web App (ambientpixelsV2)—https://calm-sky-05cc8e110.6.azurestaticapps.net/ and ambientpixels.ai.
    • Function App (ambientpixels-meme-api-fn)—API endpoints (e.g., https://ambientpixels-meme-api-fn.azurewebsites.net/api/getToken).

Why Fast? Pre-configured GitHub Actions with API tokens—Push triggers instant deployment without manual steps.

Azure Services

We run two Azure services side-by-side:

  • Static Web App (ambientpixelsV2):

    Purpose: Hosts static content—HTML, CSS, JS (e.g., template.html, _images/).

    URL: https://calm-sky-05cc8e110.6.azurestaticapps.net/—Custom domain ambientpixels.ai.

    Deployment: GitHub workflow azure-static-web-apps-calm-sky-05cc8e110.yml uploads files directly—no build step (skip_app_build: "true").

    SSL: Azure-managed HTTPS—Validated with TXT record.

  • Function App (ambientpixels-meme-api-fn):

    Purpose: Runs serverless APIs—getToken (UUIDs), memeGenerator (memes).

    URL: https://ambientpixels-meme-api-fn.azurewebsites.net/api/[endpoint].

    Deployment: GitHub workflow master_ambientpixels-meme-api-fn.yml builds Node.js functions (e.g., getToken/index.js).

    CORS: Secured with Access-Control-Allow-Origin—Allows calm-sky… and ambientpixels.ai.

Integration: Static Web App serves frontend, fetching from Function App APIs via scripts.js. Separation ensures lightweight static content and scalable APIs.

Security

API Security

Token-Based: getToken generates UUIDs—Used in memeGenerator requests (future enhancement pending).

CORS Configuration: Set in getToken/index.js:

headers: {
    "Access-Control-Allow-Origin": "https://calm-sky-05cc8e110.6.azurestaticapps.net",
    "Access-Control-Allow-Methods": "GET, POST, OPTIONS",
    "Access-Control-Allow-Headers": "Content-Type, Authorization"
}

Also configured in Azure Portal > Function App > CORS to allow ambientpixels.ai and www.ambientpixels.ai.

Status: Secured for current origins—Scalable for future domains.

Dual GitHub Workflows

Why Two Workflows?

  • Static Content: azure-static-web-apps-calm-sky-05cc8e110.yml deploys HTML/CSS/JS—Triggers on all master pushes.
  • API Functions: master_ambientpixels-meme-api-fn.yml deploys Node.js functions—Triggers on API file changes (e.g., getToken/, meme-api/, package.json).
  • Separation: Prevents unnecessary API deploys—Optimizes speed.

Static Workflow:

on:
  push:
    branches: [ "master" ]
jobs:
  build_and_deploy_job:
    steps:
      - uses: Azure/static-web-apps-deploy@v1
        with:
          azure_static_web_apps_api_token: ${{ secrets.AZURE_STATIC_WEB_APPS_API_TOKEN_CALM_SKY_05CC8E110 }}
          app_location: "/"
          output_location: ""
          skip_app_build: "true"

Function Workflow:

on:
  push:
    branches: [ "master" ]
    paths:
      - 'getToken/**'
      - 'meme-api/**'
      - 'package.json'
jobs:
  build-and-deploy:
    steps:
      - uses: Azure/functions-action@v1
        with:
          app-name: 'ambientpixels-meme-api-fn'
          package: '.'
          publish-profile: ${{ secrets.AZUREAPPSERVICE_PUBLISHPROFILE_E5BE881F58F440D69430618422EEB336 }}

Process: Single git push—GitHub splits work—Static deploys always—Function deploys conditionally.

UI Functions

toggleSection: Collapses/expands sections—Click headers.

function toggleSection(header) {
    const content = header.nextElementSibling;
    content.classList.toggle('active');
}

toggleGroup: Collapses/expands section groups—Swaps chevron icons.

function toggleGroup(header) {
    const content = header.nextElementSibling;
    content.classList.toggle('active');
    const icon = header.querySelector('.toggle-icon');
    icon.classList.toggle('fa-chevron-down');
    icon.classList.toggle('fa-chevron-up');
}

toggleAllSections: Toggles all groups and sections—Swaps expand/compress icons.

toggleMenu: Mobile navbar toggle—Burger icon.

toggleTheme: Light/dark switch—Moon/sun button.

backToTop: Snaps to top—Appears on scroll (scrollY > 200).

Notes: Intro section static—Groups (“Pipeline”, “Security”, “Style Guide”) and standalone sections centered at max-width: 1000px.
