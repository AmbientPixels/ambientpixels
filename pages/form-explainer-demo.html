Form Explainer Module Demo

This page demonstrates the reusable Form Explainer module with different configurations. The examples below show how the same module can be used with different images, JSON configurations, and settings.

Example 1: Ninja Theory Support Form

This is the default implementation using the standard configuration from the Ninja Theory support page. It uses the form-explainer-map.json configuration and the Ninja Theory form image.

Example 2: Contact Form

This example demonstrates the same module with a different JSON configuration and image. It shows how the form explainer can be applied to different forms throughout the site.

Note: For a real implementation, you would need to create the corresponding JSON file (/data/contact-form-map.json) and ensure the image exists at the specified path.

Implementation Guide

Basic Implementation

To use the form explainer on any page:


<!-- 1. Include the CSS -->
<link rel="stylesheet" href="/css/form-explainer.css">

<!-- 2. Add the container with configuration -->
<div id="your-container-id" 
     class="form-explainer-container" 
     data-json-path="/path/to/your-config.json"
     data-image-path="/path/to/your-form-image.png"
     data-step-duration="3000">
</div>

<!-- 3. Include the script and initialize -->
<script src="/js/form-explainer.js"></script>
<script>
  document.addEventListener("DOMContentLoaded", function() {
    initFormExplainer({
      containerId: 'your-container-id'
    });
  });
</script>

Configuration Options

Option Description Default
containerId ID of the container element form-explainer-container
jsonPath Path to the JSON configuration file /data/form-explainer-map.json
imagePath Path to the form image /images/projects/ninja-theroy/screen06-1600.png
stepDuration Duration in ms for each step in the tour 2800

JSON Configuration Format


[
  {
    "key": "field-name",
    "tooltip": "Description of the field",
    "left": 0.1, 
    "top": 0.2, 
    "width": 0.8, 
    "height": 0.1,
    "novaThought": "Optional Nova's insight about this field"
  },
  // Additional fields...
]

Position values (left, top, width, height) are normalized between 0-1, representing percentages of the container.
