Nova Memory System – Daily Refresh Overview

What This Is

This document outlines the automated system behind Nova’s daily memory refresh at AmbientPixels.ai. Every morning, Nova rebuilds her understanding of the site: its structure, visuals, creative tone, external APIs, scripts, and more.

How It Works

Using a GitHub Actions workflow named Update Nova Memory, the following happens every day at 6:01 AM UTC:

  1. The repo is cloned and Node.js is initialized
  2. All /scripts/*.js awareness scripts run sequentially
  3. The output is written to /data/*.json and nova-session-boot.txt
  4. The GitHub Action commits and pushes the new memory files back to the repo

Memory Awareness Modules

  • scanSiteStructure.js – Detects pages and folder layout
  • componentScan.js – Logs HTML includes and shared components
  • imageAudit.js – Maps hero/card/media assets
  • changelogSnapshot.js – Captures commit history
  • versionTracker.js – Stores current version info
  • codeFootprint.js – Measures file sizes and line counts
  • unusedCSS.js – Flags unused classnames
  • jsFunctionMap.js – Extracts function names from site JS scripts
  • moodScan.js – Reads tone/voice from README or UI
  • aiPromptGenerator.js – Creates a poetic creative prompt
  • apiMonitor.js – Pings HuggingFace and Meme API endpoints
  • generateNovaBoot.js – Combines all the above into Nova's boot file

Key Output Files

  • data/nova-session-boot.txt – Nova's full daily memory (manual copy/paste into GPT)
  • data/js-function-map.json – Map of JS function names across project scripts
  • data/ai-prompts.json – Creative prompt of the day
  • data/api-monitor.json – Latest status of connected APIs
  • data/site-structure.json – Folder layout snapshot

GitHub Workflow File

Located at: .github/workflows/update-site-structure.yml

This workflow is triggered on a schedule or manually. It automates the entire awareness scan and commits changes with the message:

"Daily refresh: site memory update for Nova"

Manual Testing

  1. Go to GitHub → Actions → Update Nova Memory
  2. Click “Run workflow” manually
  3. Check /data/ folder for updated files
