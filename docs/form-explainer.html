Form Explainer is a reusable module for interactive form guidance.

Form Explainer Module

Overview

The Form Explainer module provides interactive, step-by-step guidance for complex forms. It supports dynamic images, JSON-driven configuration, and flexible layout within the AmbientPixels grid system.

Live Demo

Component Breakdown

  • Image Container: The left column holds the form image, which serves as the backdrop for the interactive overlays.
  • Field Stack: The right column displays a list of all form fields, driven by your JSON configuration.
  • Numbered Headlines: Each step's title is dynamically updated with a number and content from the JSON map.
  • Automated Tour: The module automatically cycles through each step, highlighting the corresponding field and overlay.
  • Responsive Design: The layout adapts seamlessly from a two-column view on desktops to a single-column stack on mobile devices.

Basic Usage

To use the Form Explainer module, include the following markup in your page:

<div class="form-explainer" data-config="/data/form-explainer-map.json">
  <!-- Module will auto-initialize and render here -->
</div>
<script src="/js/form-explainer.js" defer></script>

Make sure to load the required CSS and JS files, and provide a valid JSON config.

Example JSON Configuration

The module reads its steps and content from a JSON file. Example:

{
  "title": "Support Request",
  "steps": [
    { "label": "Your Name", "type": "text", "placeholder": "Enter your name" },
    { "label": "Email", "type": "email", "placeholder": "Enter your email" },
    { "label": "Describe your issue", "type": "textarea" }
  ],
  "image": "/images/support-form.jpg"
}

Place your config in /data/form-explainer-map.json or another path, and reference it via data-config.

Integrating with the Site Grid

Use grid-col-6, grid-col-12, or other grid classes to position the module within your layout. Example:

<section class="content-section grid-col-6">
  <div class="form-explainer" data-config="/data/form-explainer-map.json"></div>
</section>

Styling & Customization

  • Customize appearance via form-explainer.css and global theme tokens (--aura-*, --mood-*).
  • Use existing utility classes for spacing, mood, and layout.
  • For advanced use, extend the module with custom JS and additional config options.
